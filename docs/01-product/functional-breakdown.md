# ResumeGenius 功能划分

更新时间：2026-04-22

## 1. 功能划分原则

- 项目本质是一个简历编辑 Agent，而不是传统模板站或纯编辑器。
- 功能应按业务闭环划分，不按前后端或模型能力横向划分。
- Agent 是主入口，人工工作台是辅助入口，但两者共享同一份简历状态。
- v1 demo 目标是跑通“生成简历 + 修改简历 + 导出 PDF”的完整闭环。

## 2. 一级功能域

### 2.1 资料接入与证据抽取

职责：

- 接收用户输入资料
- 统一抽取结构化证据
- 为初始简历生成提供输入

支持输入：

- 简历文件：PDF / DOCX / PNG / JPG
- 补充文本资料
- Git 仓库

产出：

- `SourceAssetSet`
- `EvidenceSet`

### 2.2 初始简历生成

职责：

- 当用户已有简历时，从原简历重构首版
- 当用户没有简历时，从资料集合生成首版
- 生成可继续编辑的初始简历状态

产出：

- `ResumeDraftState`

### 2.3 对话式 Agent 编辑

职责：

- 解析用户自然语言需求
- 识别内容修改、样式修改或混合修改
- 产出结构化 Patch 或建议

支持能力：

- 内容改写
- 内容压缩
- section 重排建议
- 样式调整建议
- 混合修改

产出：

- `PatchEnvelope(source=agent)`

### 2.4 人工工作台编辑

职责：

- 提供一个轻量工作台用于直接修改内容和样式
- 不做完全自由排版，不暴露底层 TeX

v1 支持：

- 内容编辑：section 文本、bullet、顺序、显隐
- 样式编辑：字体大小、样式主题、行距、字距、段距

产出：

- `PatchEnvelope(source=manual)`

### 2.5 状态管理与版本控制

职责：

- 应用 Patch
- 校验修改是否合法
- 记录版本
- 支持撤销和回退

产出：

- 新的 `ResumeDraftState`
- `RevisionRecord`

### 2.6 预览、渲染与导出

职责：

- 生成用户可见预览
- 将编辑态转换为渲染态
- 编译并导出 PDF

产出：

- `ResolvedResumeSpec`
- PDF

## 3. v1 demo 必做范围

### 3.1 必须做

- 新建项目
- 上传文件
- 接入 Git 仓库
- 补充文本资料
- 证据抽取
- 初始简历生成
- Agent 内容修改
- Agent 样式修改
- 人工工作台内容修改
- 人工工作台样式修改
- Patch 应用与回退
- 预览
- PDF 导出

### 3.2 可以简化

- Git 仓库首版只抽 README、项目名、技术栈、提交摘要、关键源码目录说明
- 人工工作台不做真正所见即所得富文本编辑器
- 样式系统首版只做有限参数，不做自由拖拽
- 预览首版允许“准实时”，不强求毫秒级同步

### 3.3 明确不做

- 原生 TeX 编辑
- 完全自由拖拽排版
- 投递系统
- 草图导入
- VLM 主链路
- AI 直接决定最终精排数值

## 4. 推荐的 5 人模块切分

### 模块 A：资料接入

职责：

- 项目创建
- 文件上传
- Git 仓库接入
- 资料存储

输入输出：

- 输入用户资料
- 输出 `SourceAssetSet`

### 模块 B：证据抽取与初稿生成

职责：

- PDF / DOCX / 图片解析
- Git 仓库信息抽取
- 证据归一化
- 初始简历生成

输入输出：

- 输入 `SourceAssetSet`
- 输出 `EvidenceSet` 和初始 `ResumeDraftState`

### 模块 C：Agent 编辑链路

职责：

- 对话会话
- 意图识别
- 调用内容/样式能力
- 生成 Agent Patch

输入输出：

- 输入用户自然语言 + 当前草稿 + 证据
- 输出 `PatchEnvelope(source=agent)`

### 模块 D：人工工作台

职责：

- 内容编辑 UI
- 样式编辑 UI
- 工作台交互
- 生成 Manual Patch

输入输出：

- 输入当前草稿
- 输出 `PatchEnvelope(source=manual)`

### 模块 E：状态、预览、渲染、导出

职责：

- Patch 应用
- 版本记录
- 预览生成
- 渲染解算
- LaTeX/PDF 导出

输入输出：

- 输入 Patch + 当前草稿
- 输出新草稿、预览、PDF

## 5. 当前最应该先冻结的接口

建议先冻结下面 4 组协议，再进入详细实现：

1. `SourceAssetSet / EvidenceSet`
2. `ResumeDraftState`
3. `PatchEnvelope`
4. `ResolvedResumeSpec`
