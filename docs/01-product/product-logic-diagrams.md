# ResumeGenius 产品功能关系与用户逻辑图

更新时间：2026-04-22

本文档保存当前已经确认的系统关系图与用户流程图，作为后续功能拆分、技术架构和协议设计的基础。

## 1. 功能关系图

```mermaid
flowchart LR
    A[用户] --> B[对话式 Agent]
    A --> C[人工修改工作台]

    D[资料输入]
    D --> D1[文件上传]
    D --> D2[Git 仓库接入]
    D --> D3[补充文本资料]

    D1 --> E[资料解析与证据抽取]
    D2 --> E
    D3 --> E

    E --> F[SourceAssetSet/证据库]
    F --> G[初始简历生成]

    G --> H[统一简历状态]

    B --> I[Agent 编排层]
    I --> J[内容能力]
    I --> K[样式能力]
    J --> L[AI Patch/Suggestion]
    K --> L

    C --> M[Manual Patch]
    C --> N[内容编辑]
    C --> O[样式编辑]

    H --> P[Patch 应用与校验]
    L --> P

    P --> Q[新版本简历状态]

```

## 2. 用户使用逻辑图

```mermaid
flowchart TD

    D --> E{用户是否已有简历}

    E -- 没有 --> G[提供相关资料]

    G --> G1[上传文件]
    G --> G2[连接 Git 仓库]
    G --> G3[填写补充信息]

    F --> H[解析与抽取]
    G1 --> H
    G2 --> H
    G3 --> H

    H --> I[形成证据库/资料集合]
    I --> J[生成初始简历]
    C --> K[加载当前简历状态]

    K --> L

    L --> M{选择修改方式}

    N --> P[Agent 理解需求]

    W --> Y[应用 Patch]

    Y --> Z[更新预览]

    Z --> AA{是否继续修改}
    AA -- 是 --> M
    AA -- 否 --> AB[导出 PDF]
```

## 5. 当前已确认的结论

- 项目本质是一个简历编辑 Agent，Agent 是主入口。
- 人工修改工作台是辅助入口，但也同时支持内容和样式修改。
- 用户可以有简历，也可以没有简历。
- 用户输入资料支持文件、Git 仓库和补充文本资料。
- 内容修改与样式修改是能力分类，不是页面分类。
- Agent 和人工工作台最终都应收敛到统一简历状态。
- 统一 Patch 机制是后续协议设计的核心。
- 最终输出链路应统一为预览、渲染解算、PDF 导出。
