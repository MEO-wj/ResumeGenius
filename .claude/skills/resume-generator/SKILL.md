# Resume Generator — 中文技术简历生成器

## 概述

生成一份专业的单栏学术风中文技术简历，LaTeX (ctexart) 格式，编译为 1 页 A4 PDF。

核心目标：让面试官在 30 秒内得出三个结论——基础扎实、能独立负责项目、能带来业绩提升。

## 工作流程

### Phase 1: 信息收集

向用户逐项收集信息。对于已有的信息（文件中已有、用户已提供），跳过确认即可。

必选项：
1. **基本信息**：姓名、学校、专业、学历、GPA/排名、英语等级、联系方式
2. **目标岗位**：决定项目描述的侧重点（AI方向突出工程链路闭环，后端方向突出架构设计）
3. **照片**：是否放照片，照片文件名
4. **经历**：实习/项目/竞赛，按 STAR 格式描述
5. **技能**：分类呈现（后端/AI/前端）

可选输入文件：
- 个人信息文件（如 `个人信息.md`）
- 实习经历文件（如 `实习经历.md`）
- 项目源码目录（可分析提取技术细节）
- 简历要求文件（如 `简历要求.md`，定义写作标准和量化要求）

### Phase 2: 内容撰写

#### 核心句式

每条经历遵循：**动词 + 技术方案 + 效果**

```
❌ 实现了 RAG 检索功能
✅ 针对单一向量检索召回不全的问题，设计四级复合检索流水线（向量→模糊兜底→重排序→按需提取），保持上下文干净

❌ 负责后端 API 开发
✅ 基于 PostgreSQL LISTEN/NOTIFY 构建事件驱动的 Embedding 流水线，文章发布即触发分块→向量化→批量入库
```

#### STAR 法则

- **Situation**: 业务背景/痛点（一句话）
- **Task**: 你的职责/目标（可选，融入 Action）
- **Action**: 技术方案 + 为什么选它（重点）
- **Result**: 量化效果（必须带数字）

如果用户没有量化数据，先按最佳实践写出结构，用 `[待补充]` 标记需要数据的位置，让用户补充。

#### AI 项目写作要点

体现**工程链路闭环**，而非"调包侠"：
- RAG：检索策略设计（为什么多层？每层解决什么问题？）、切片策略、重排序
- Agent：可靠性设计（harness/流程控制）、工具编排、意图理解
- 成本/准确率：量化指标（召回率、幻觉率、Token 消耗）
- 部署状态：真实用户数、是否上线

#### 后端项目写作要点

体现**架构设计**，而非"CRUD"：
- 高并发：限流设计、队列管理、并发模型
- 数据：向量存储、自动化流水线、类型安全
- 可维护：热插拔、代码生成、配置管理

#### 项目排列规则

1. 有真实用户的部署项目 > 有完整技术链路的项目
2. 与目标岗位最匹配的项目排最前
3. 重点项目 3 个 bullet，次要项目 1 个 bullet

#### 技能呈现

分类呈现（后端/AI/前端），用括号标注核心框架，避免流水账：
```
后端：Go（Gin/GORM/sqlc），Python（FastAPI），PostgreSQL（pgvector），Docker
AI：RAG，Function Calling / Agent 架构，harness 工程
前端：React / React Native，Vue 3，TypeScript
```

### Phase 3: 生成 LaTeX

读取 `references/template.tex` 获取基础模板，根据收集的信息填充内容。

关键格式规范：
- 个人信息栏使用 minipage 布局（左文字 73% + 右照片 22%）
- 项目标题使用 `\roleheader{项目名}{}`
- 技术栈行使用 `{\small\color{lightgray} ...}`
- 实习 header：公司+岗位放一行，地点+日期放第二行灰色小字（节省空间）
- 内容使用 `\begin{itemize} \item ... \end{itemize}`
- **项目之间不使用空行**，用 `\vspace{0.1em}` 控制间距

### Phase 4: 压缩到一页

**新生成的简历几乎总是会超出一页。** 只调整排版，不修改内容。

按优先级依次尝试，直到内容 fit 一页：

| 优先级 | 方法 | 预期效果 |
|--------|------|---------|
| 1 | `\linespread{1.15}`（ctexart 默认 1.3） | 全局行距压缩约 15%，节省 4-5 行 |
| 2 | 边距缩减 `margin=1.1cm, top=0.7cm, bottom=0.7cm` | 节省 2-3 行 |
| 3 | `\titlespacing*{\section}{0pt}{0.25em}{0.05em}` | 每个节标题节省约 0.3em |
| 4 | `\setlist[itemize]{itemsep=0pt}` | 每 10 个 bullet 节省约 1 行 |
| 5 | 合并实习 header（地点+日期用灰色小字第二行） | 节省 2 行 |
| 6 | 删除所有 section 之间的空行 | 节省 1-2 行 |
| 7 | 减少个人信息栏 `\\[Xem]` 间距 | 节省 0.5-1 行 |
| 8 | 进一步缩减边距 `margin=1.0cm, top=0.6cm` | 节省 1-2 行 |

### Phase 5: 编译验证

```bash
xelatex resume.tex
```

检查清单：
- [ ] 恰好 1 页
- [ ] 所有标题顶格（无缩进）
- [ ] 照片与文字对齐
- [ ] 无内容溢出/截断

## 常见坑与解决方案

### 1. 项目标题不顶格
**原因**：`\end{itemize}` 后的空行产生 LaTeX 段落缩进
**方案**：preamble 加 `\setlength{\parindent}{0pt}` 全局禁用段落缩进；项目间不使用空行，用 `\vspace` 控制间距

### 2. 照片与文字不对齐
**原因**：minipage 使用 `[t]` 对齐或内部使用 `\newline`
**方案**：用 `[c]` 居中对齐；minipage 内用 `\\` 换行（不是 `\newline`）；照片 minipage 用 `\raggedleft`

### 3. 奖项/联系方式被截断
**原因**：强行合并到一行导致自动换行后割裂
**方案**：保持多行格式，不要为了省空间强行压缩联系信息

### 4. 内容超出一页
**原因**：默认 ctexart 行距（1.3）和间距对简历太宽松
**方案**：按 Phase 4 的优先级顺序系统性压缩。先全局参数（linespread、边距），再局部间距（section spacing、itemsep），最后是微调（vspace、header间距）

### 5. section 横线分割感太强
**方案**：减少 `\titlespacing` 的 before/after 值到 `{0pt}{0.25em}{0.05em}`

### 6. 内容修改后页数变化
**方案**：内容修改和排版调整分开进行——先改内容，再系统性压缩排版

## 编译依赖

- TeX Live 或 MiKTeX
- ctex 宏包（中文支持）
- fontawesome5 宏包（图标）
